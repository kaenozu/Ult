# Quality Gatekeeper Skill (品質ゲートキーパー)

このスキルは、コードの変更がシステム全体に悪影響を与えていないか、またユーザーに提供するデータが論理的に正しいかを最終確認するための「検問」として機能します。

## 1. 完了前のセルフチェックリスト
作業を「完了」とする前に、以下の項目を必ず実行し、結果を確認します。

### A. 機械的チェック (Build & Type Check)
*   `npm run build` または `tsc` を実行し、ファイル間の重複定義、型エラー、インポートミスがないか確認する。
*   特に関連ファイルが多い修正（銘柄追加など）の後は必須とする。

### B. 視覚・論理的チェック (Visual Anomaly Detection)
ブラウザのスナップショットを取得し、以下の「違和感」を能動的に探す：
*   **計算の異常**: 騰落率 100% 超え、資産の急増、負の数量など（単位ミスや 100倍重複の検知）。
*   **データの欠損**: `NaN`, `undefined`, `null`, `Infinity` がUIに露出していないか。
*   **レイアウトの崩れ**: 文字溢れ、ボタンの重なり、ローディングの永続化。

### C. 言語の一貫性
*   新しい項目が英語のままになっていないか（日本語化の徹底）。

## 2. 異常発見時のプロトコル
*   異常を発見した場合は「完了」を報告せず、即座に修正ブランチ（または現在の作業）で修正を行い、再度ゲートキーパーを実行する。
