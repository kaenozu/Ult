# Smart Git Skill (スマート・ギット)

このスキルは、複雑なコンポーネント分割やリファクタリングが行われたコードベースにおいて、複数の機能ブランチを矛盾なくマージし、品質を維持するための高度なGit運用を定義します。

## 1. 競合の論理的解決 (Logic Transplantation)
単なるテキストの衝突解決ではなく、リファクタリングによる「居場所の変化」を検知し、ロジックを正しく移植する。
*   **例**: `page.tsx` にあったフェッチロジックが `useStockData.ts` フックに移動していた場合、マージ対象の最適化ロジックもフック側へ自動的に適用する。

## 2. マージ後の品質同期 (Post-Merge Synchronization)
マージ直後に必ず以下の3点を確認し、不整合を自動修正する：
*   **インポート整合性**: コンポーネント分割に伴うパスの変更や、デフォルト/名前付きインポートの不一致を即座に修正する。
*   **型定義の同期**: 型の拡張（プロパティ追加など）が全コンポーネントに波及しているか確認し、ビルドエラーを未然に防ぐ。
*   **テストの言語同期**: UIの日本語化とテストコード内のアサーション（検索テキスト）が一致しているか確認し、偽陽性の失敗を排除する。

## 3. プルリクエスト・オーケストレーション
複数の PR を処理する際は、以下のフローを遵守する：
1.  依存関係（構造変更など）の大きい PR を優先的にマージ。
2.  残りの PR に対して `main` をマージ（またはリベース）し、個別に競合を解決。
3.  最終的に「ビルドエラー・テスト失敗ゼロ」の状態を GitHub に反映する。
