# Git Auto-Survival Skill (Gitオート・サバイバル)

このスキルは、Git操作中に発生するロック問題や対話プロンプトを自律的に解決し、スムーズな同期とプッシュを維持するための手順を定義します。

## 1. ロック問題の自律解決
Git操作中に `index.lock` やファイルロック（Unlink failed）が発生した場合、以下の手順を即座に実行する：
*   **物理ロック解除**: `rm -Force .git/index.lock` を実行する。
*   **プロセスロック解除**: サーバー（ポート3000）を `safe-process-manager` の手順で停止し、ログファイル等のロックを解放する。

## 2. 非対話実行の徹底
`git clean` や `git reset` など、ユーザー入力を求める可能性があるコマンドを実行する際は、プロンプトを回避するフラグ（`-f`, `-d`, `-q` 等）を必ず使用する。ユーザーに「y/n」を答えさせる状況を作らない。

## 3. 強制同期プロトコル
ローカルとリモートが大幅に乖離し、通常のプルが困難な場合は、以下の「クリーン・同期」を迷わず選択する：
1.  サーバー停止。
2.  `git reset --hard origin/main`
3.  `git clean -fd`
4.  `git pull origin main`

## 4. gitignore の遵守
ログファイル、キャッシュ、環境変数などが Git 追跡に含まれている場合は、即座に `git rm --cached` を行い、`.gitignore` を更新して再発を防止する。
