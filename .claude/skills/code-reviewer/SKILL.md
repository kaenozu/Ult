# Code Reviewer スキル

コードの完全性、セキュリティ、パフォーマンスを総合的にレビューします。

## レビュー項目

### 1. TypeScript型安全性
- `strict`モード違反がないか
- `any`型の使用がないか
- 型定義の適切性

### 2. コード品質
- ESLint違反
- 未使用コード
- 複雑度（cyclomatic complexity）
- 重複コード

### 3. セキュリティ
- インジェクション脆弱性（XSS, SQLi）
- 認証・認可の問題
- シークレットリーク
- 入力検証不足

### 4. テストカバレッジ
- 新規コードにテストがあるか
- エッジケース対応
- モック適切性

### 5. パフォーマンス
- O(N²) または O(N³) 算法
- 大规模データ処理時の問題
- メモリリーク可能性
- 重たい処理を同期実行していないか

## レビュー報告形式

```
## 📋 コードレビュー結果

### ✅ 良好点
- [良い点を列出]

### ⚠️ 懸念事項
- [問題点と修正提案]

### 🔧 推奨修正
1. [具体的な修正手順]
2. [優先度: 高/中/低]

### 📊 スコア
セキュリティ: A/B/C/D/F
パフォーマンス: A/B/C/D/F
保守性: A/B/C/D/F
```

## 自動適用条件

ユーザーが以下のいずれかを発言：
- 「レビューして」「コードレビューして」
- 「確認して」「チェックして」「検証して」
- ファイル変更後の自動レビュー（CLAUDE.md Setting）
