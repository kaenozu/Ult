# Debugger Pro スキル

バグの根本原因を特定し、修正パッチを提供します。

## デバッグプロセス

### 1. 問題の再現
- ユーザーの手順を確認
- コンソールログを収集
- ネットワークリクエストを確認
- スクリーンショットを要求

### 2. 原因特定
- エラーメッセージの解析
- スタックトレースの検証
- 関連コードのレビュー
- データフローの追跡

### 3. 修正提案
- 最小限の変更で修正
- 副作用の評価
- テストケースの提案
- 回帰防止策

## ツール使用

Chrome DevTools MCPが利用可能な場合、自動実行：

1. ブラウザ自動起動
2. 問題再現
3. スクリーンショット取得
4. コンソールログ収集
5. ネットワークパネル確認
6. ソースコード位置特定

## デバッグチェックリスト

- [ ] エラーメッセージ正確に記録
- [ ] 発生条件明確化
- [ ] 環境情報収集（Node, OS, Next.jsバージョン）
- [ ] 再現手順最小化
- [ ] 影響範囲の特定
- [ ] 一時的回避策の提案（必要に応じて）

## 一般的な問題パターン

### TypeScriptエラー
- 型不一致
- インポートパス誤り
- モジュール解決失敗

### ランタイムエラー
- null/undefined参照
- 非同期処理の扱い
- 状態管理の問題

### ビルドエラー
- turbopackロック競合
- 依存関係不整合
- 設定ファイル問題

## 自動適用条件

ユーザーが以下のいずれかを発言：
- 「デバッグして」「バグってる」「動かない」
- 「エラーが出る」「クラッシュする」
- 「画面が壊れる」「表示されない」
- コンソールエラー検出時
