# Security Checklist for Developers

コードレビュー時や新規実装時に確認すべき項目です。

## 🔑 秘密情報と認証
- [ ] `JWT_SECRET` が32文字以上であることをテストで確認しているか？
- [ ] APIキーや秘密情報をログに出力していないか？
- [ ] 認証が必要なルートに `requireAuth` または `requireAdmin` が適用されているか？

## 🌐 通信とヘッダー
- [ ] `next.config.ts` で `X-Frame-Options: DENY` が維持されているか？
- [ ] 外部ドメインへのリクエストに適切なタイムアウトが設定されているか？
- [ ] 入力値はすべて Zod 等でバリデーションされているか？

## 🚨 エラーハンドリング
- [ ] エラーメッセージにスタックトレースや内部パスなどの機密情報が含まれていないか？
- [ ] 統一エラークラス（`AppError` サブクラス）を使用しているか？
- [ ] APIレスポンスは `handleApiError` を通じて生成されているか？
